# LiteLLM Gateway — 技术设计文档

## 1. 技术选型

### 选型决策表

| 维度 | 选择 | 理由 |
|------|------|------|
| 网关引擎 | **LiteLLM** | 48k⭐、100+ provider、原生 cost-based routing、多协议、模型别名 |
| 运行模式 | **config-only** | 零数据库依赖，笔记本友好 |
| 日志存储 | **SQLite** | LiteLLM 内置 `litellm-db`，自动管理 |
| 部署方式 | **systemd user service** | 开机自启、崩溃自愈、不依赖 Docker |
| 模型探索 | **Python 脚本 + cron** | 灵活、可扩展 |
| 配置热加载 | **SIGHUP / API** | LiteLLM 原生支持 |

```yaml
  routing_strategy: cost-based-routing  # 成本优先

  set_verbose: false                    # 生产关掉 verbose
```

## 4. auto-discover 设计

### 4.1 架构

```
┌─────────────────────────────────────────────────────┐
│                 discover.py                          │
│                                                     │
│  1. 读取上次快照 (discover_cache.json)               │
│  2. 并行扫描各平台                                    │
│     ├─ OpenRouter: GET /api/v1/models                │
│     ├─ Groq: GET /v1/models                          │
│     └─ 扩展点：Gemini/GitHub Models 等               │
│  3. 过滤出免费模型 (price == 0)                       │
│  4. 与快照对比，计算差集：                              │
│     ├─ NEW: 新增免费模型                              │
│     ├─ GONE: 消失的模型                               │
│     └─ PRICE_CHANGED: 价格变动（free→paid!）          │
│  5. 如果有变化：                                      │
│     ├─ 更新 LiteLLM config.yaml                       │
│     ├─ SIGHUP LiteLLM 重载配置                        │
│     ├─ 更新 discover_cache.json                       │
│     └─ 写入 discover_changelog.log 通知用户           │
└─────────────────────────────────────────────────────┘
```

### 4.2 运行参数

| 参数 | 值 |
|------|-----|
| 执行频率 | 每日 08:00（cron） |
| 超时 | 60 秒 |
| 缓存文件 | `~/.hermes/data/discover_cache.json` |
| 变更日志 | `~/.hermes/data/discover_changelog.log` |

### 4.3 通知格式

```
📢 免费模型更新报告
━━━━━━━━━━━━━━━━━━
🆕 新增: openrouter/xxx:free (供应商: xxx)
❌ 消失: groq/yyy (已下架)
💰 变价: zzz (Free → Paid!) ⚠️ 自动摘除
━━━━━━━━━━━━━━━━━━
本次扫描: 45 个模型 (新增 2, 消失 1)
```

## 5. 部署方案

### 5.1 前置条件

```bash
# 1. Python 3.10+
python3 --version

# 2. 安装 LiteLLM
pip install 'litellm[proxy]'

# 3. Ollama（可选）
curl -fsSL https://ollama.com/install.sh | sh
ollama pull qwen2.5:1.5b

# 4. 配置 .env
cat > config/.env << EOF
LITELLM_MASTER_KEY=sk-your-master-key
OPENROUTER_API_KEY=sk-or-xxx
DEEPSEEK_API_KEY=sk-xxx
ZHIPU_API_KEY=xxx
GROQ_API_KEY=sk-groq-xxx
EOF
```

### 5.2 启动方式

```bash
# 前台测试
litellm --config config/config.yaml --port 4000

# systemd 用户服务（持久化）
# 见 systemd 配置节
```

### 5.3 systemd 服务

```ini
# ~/.config/systemd/user/litellm-gateway.service
[Unit]
Description=LiteLLM AI Gateway
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=/usr/bin/litellm --config /home/sunzc/Documents/AIProjects/litellm-gateway/config/config.yaml --port 4000
WorkingDirectory=/home/sunzc/Documents/AIProjects/litellm-gateway
EnvironmentFile=/home/sunzc/Documents/AIProjects/litellm-gateway/config/.env
Restart=always
RestartSec=10
ExecReload=/bin/kill -SIGHUP $MAINPID

[Install]
WantedBy=default.target
```

```bash
systemctl --user daemon-reload
systemctl --user enable --now litellm-gateway
```

### 5.4 Cron 配置

```bash
# 每日 08:00 运行探索
0 8 * * * cd /home/sunzc/Documents/AIProjects/litellm-gateway && python3 scripts/discover.py >> /home/sunzc/.hermes/data/discover_changelog.log 2>&1
```

## 8. 错误处理

| 错误场景 | 表现 | 处理方式 |
|----------|------|---------|
| 所有免费模型不可用 | 429 | 不降级到付费模型，直接返回错误 |
| OpenRouter API 不可用 | fallback | 自动切到 DeepSeek 直连 |
| .env 缺失 | 启动失败 | 日志提示缺少的变量 |
| config.yaml 格式错误 | 启动失败 | LiteLLM 给出具体 YAML 错误信息 |
| 探索脚本扫描失败 | 跳过该平台 | 日志记录，不影响其他平台 |

## 9. 扩展设计

### 9.1 新 Provider 接入

在 `scripts/discover.py` 中新增扫描器：

```python
class GroqScanner(FreeModelScanner):
    """Groq：全部模型免费（限速 30 RPM）"""
    name = "groq"
    api_url = "https://api.groq.com/openai/v1/models"

    def scan(self) -> list[Model]:
        # 所有模型都是免费的
        resp = self._get(self.api_url)
        return [Model(id=m["id"], provider="groq", price=0)
                for m in resp["data"]]
```

注册到主扫描器即可，无需修改核心逻辑。

## 10. 已知风险

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| LiteLLM 上游出现破坏性变更 | 中 | 锁定版本，不自动升级 |
| OpenRouter:free 模型不稳定 | 中 | 多模型 fallback |
| 探索脚本遗漏新平台 | 低 | 模块化设计，加平台很简单 |
| SQLite 日志无限增长 | 低 | 添加日志轮转（30 天清理） |
