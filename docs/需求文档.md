# LiteLLM Gateway — 免费保险模型中转站

## 项目概述

基于 LiteLLM 搭建的本地 AI 模型中转站，聚合多家厂商的免费 API 额度，提供统一接入、自动探索、无缝切换、用量统计等能力。跑在个人笔记本上，轻量、低成本、零维护。

---

## 背景与痛点

目前已有多个 AI 模型供应商：

| 供应商 | 模型 | 性质 | 状态 |
|--------|------|------|------|
| OpenRouter | deepseek-v4-flash:free / gemma-4:free | 免费配额 | ✅ 常态使用 |
| DeepSeek 直连 | deepseek-chat | ¥25.55 余额 | ✅ 备选 |
| 智谱 | GLM-4V-Flash | 免费 | ✅ 视觉任务 |
| Groq | 全系列 | 免费限速 | 待接入 |
| Gemini AI Studio | Flash/Gemma 系列 | 免费额度 | 待接入 |
| 本地 Ollama | Qwen2.5-1.5B | 免费本地 | ✅ 轻量任务 |

**痛点：**
1. 各供应商独立管理，配置分散
2. 没有统一拦截机制，可能误调用付费模型（已发生过）
3. 免费模型不定期变化（新增/消失/转付费），需要人工跟踪
4. 无法统一观察用量和费用

---

## 核心功能

### 功能一：🕵️ 免费模型自动探索

**目标：** 自动发现各平台新增的免费模型，无需手动跟踪。

**覆盖范围：**
- OpenRouter：扫描 `price=0` 的模型
- Groq：全部免费模型
- Gemini AI Studio：免费额度内模型
- 社区维护的 free-llm-api-resources 列表

**行为：**
- 每日定时扫描一次
- 发现新免费模型 → 自动注册到路由池
- 发现模型转付费 → 自动摘除 + 通知
- 发现模型消失 → 自动摘除

### 功能二：🔄 无缝切换

**目标：** 同一个模型名背后挂多个实际模型，自动选最优可用的。

**策略：**
- 主策略：**cost-based routing** — 优先最便宜的模型
- 兜底：**fallback 链** — 首选挂了自动切下一个
- 热加载：发现新模型后不用重启进程，SIGHUP 重载配置

**模型别名映射（对外统一）：**

| 对外名 | 用途 | 实际模型池 |
|--------|------|-----------|
| `smart` | 日常对话/推理 | gemma-4:free → groq/llama → deepseek直连 |
| `fast` | 快速响应 | groq/llama-3.1-8b → openrouter 快速模型 |
| `coder` | 编程任务 | 待定 |
| `vision` | 看图分析 | 智谱 GLM-4V-Flash → OpenRouter 多模态 |
| `local` | 轻量本地摘要 | Ollama Qwen2.5-1.5B |

### 功能三：🛡️ 付费保险

**目标：** 零意外付费。

- `max_budget = $0` 硬上限拦截任何可能产生费用的调用
- 所有模型必须显式登记在 config 中，不落未知通道
- 配置修改前做 diff 检查，防止误放行付费模型

### 功能四：🏷️ 模型名桥接

**目标：** 调用方永远面对统一模型名，不感知后端模型变更。

- LiteLLM 原生 `model_name` 机制
- 同一个别名的多个模型自动做成本排序和健康检测
- 新增/移除后端模型不影响客户端

### 功能五：📊 日志与用量统计

**目标：** 完整记录每次调用，支撑用量分析和问题排查。

| 记录项 | 说明 |
|--------|------|
| 时间戳 | 调用时间 |
| 请求模型名 | 客户端传的别名（如 `smart`） |
| 实际模型 | 路由到的具体模型 |
| 供应商 | OpenRouter/Groq/智谱等 |
| Token 数 | input/output 分别计 |
| 响应耗时 | ms 级 |
| 费用 | 始终为 $0（免费保险） |
| 状态 | 成功/失败/限流/超时 |
| 错误信息 | 失败时记录 |

存储：SQLite 本地文件，轻量无依赖。

### 功能六：🔌 多协议支持

- **OpenAI 协议** (`/v1/chat/completions`) — 主入口
- **Anthropic 协议** (`/v1/messages`) — 兼容 Claude 系客户端
- 内部自动做协议转换

### 功能七：🔑 Key 保护

- 真实 API Key 仅配置在服务器端，不暴露给任何客户端
- LiteLLM 虚拟 Key 机制，可为不同使用方生成带权限限制的子 Key

### 功能八：📸 多模态

- 图片输入透传（不拦截、不转换 body）
- 支持的模型：智谱 GLM-4V-Flash、Gemini Flash、OpenRouter 多模态模型

### 功能九：🖥️ 本地模型

- 对接 Ollama ，本地 Qwen2.5-1.5B 列入模型池
- 可在路由策略中排在首位（优先免费本地）

## 约束条件

| 约束 | 说明 |
|------|------|
| 硬件 | 个人笔记本（GTX 1050 4GB） |
| 存储 | SQLite 本地文件，不跑 Postgres |
| 部署 | Docker 或直接 Python 进程 |
| 网络 | 国内环境，需可访问境外 API |
| 安全 | 仅监听 localhost，不暴露公网 |
| 维护 | 零手动维护，自动化兜底 |
